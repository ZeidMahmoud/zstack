# zstack

zstack is a personal rebrand and snapshot of [gstack](https://github.com/garrytan/gstack),
bundled together with the rest of the Claude Code workflow it runs inside. It exists so
the whole setup, the skill suite **plus** the plugins and installed skills that surround
it, lives in one repo that one person owns and can re-deploy anywhere.

Everything that read `gstack` / `GSTACK` / `GStack` / `Gstack` now reads
`zstack` / `ZSTACK` / `ZStack` / `Zstack`: file contents, file names, directory names,
CLI binary names (`bin/zstack-*`), config paths (`~/.zstack/`, `~/.zstack-dev/`), env
vars (`ZSTACK_*`), and skill names (`/open-zstack-browser`, `zstack-upgrade`, etc.).

> Provenance, in plain terms: the upstream tool, its philosophy (`ETHOS.md`), and the
> founder voice throughout the skill prose are Garry Tan's work on gstack. zstack is a
> local fork for personal use, not a competing product or a claim of authorship over
> that material. If you push this anywhere public, keep the MIT `LICENSE` and the
> attribution intact.

## Layout

```
zstack/
├── <skill dirs>/        # the full skill suite (ship, review, qa, cso, spec, ...)
├── browse/  design/     # the power-tool binaries (Playwright browser, GPT-Image design)
├── bin/                 # zstack-* CLI utilities (renamed from gstack-*)
├── hosts/ scripts/      # host configs + the gen-skill-docs build pipeline
├── setup                # one-time install: builds binaries + symlinks skills
├── ZSTACK.md            # this file
└── claude-extras/       # the rest of my Claude Code workflow (see below)
```

## The workflow extras (`claude-extras/`)

These are the "extra things from my Claude `skills/` and plugins" captured alongside the
core tool, so a fresh machine gets the same environment.

### Plugins (`claude-extras/plugins/`)

A full copy of `~/.claude/plugins/` (~9 MB). Installed from the
`claude-plugins-official` marketplace:

| Plugin | What it adds |
|--------|--------------|
| `superpowers` (5.1.0) | The skill-discipline layer (`using-superpowers`, brainstorming, TDD, systematic-debugging, writing-plans, worktrees, code-review flows). |
| `frontend-design` | Distinctive, production-grade UI generation that avoids generic AI aesthetics. |
| `context7` | Live library/framework docs on demand (MCP server). |
| `code-review` | PR / diff review slash commands. |
| `github` | GitHub PR + issue tooling. |
| `clangd-lsp` (1.0.0) | C/C++ language-server integration. |

The marketplace cache and manifests (`installed_plugins.json`,
`known_marketplaces.json`, `marketplaces/`) are included so the set is reproducible.

### Installed skill registrations (`claude-extras/skills/`)

A snapshot of the other 54 entries in `~/.claude/skills/`. Each is a thin wrapper whose
`SKILL.md` symlink points at the live install — they are the **registration layer**, not
independent content, and the actual skill bodies live in this repo's top-level dirs. Kept
for reference / reinstall; the symlink targets still reference the original install path,
so re-run `./setup` to regenerate them against zstack.

## Install on a new machine

```bash
cd zstack
./setup                       # builds browse/design binaries, symlinks skills into ~/.claude/skills

# restore the plugins (optional, only if you want the same plugin set)
cp -a claude-extras/plugins/. ~/.claude/plugins/
```

`./setup` reads `skill_prefix` from `~/.zstack/config.yaml` (pass `--no-prefix` or
`--prefix` to skip the prompt). After a `git pull`, re-run `./setup`.

## What was NOT changed

- **Compiled binaries** under `*/dist/` — arm64 macOS Mach-O, regenerated by `./setup`
  on each platform, so they were left untouched (they don't run on this Linux box anyway).
- **`node_modules/`** — vendored dependencies, not rebranded.
- **Founder-voice prose and `ETHOS.md`** — left intact except for the literal brand token.

## Building / testing

Same toolchain as upstream (now branded zstack):

```bash
bun install
bun test                 # free tests (browse + snapshot + skill validation)
bun run build            # gen docs + compile binaries
bun run gen:skill-docs   # regenerate SKILL.md files from templates
```

See `CLAUDE.md` for the full command reference and contributor rules.
